# 首版前端架构说明

## 目标

这份文档说明 `mini-home` 当前这版前端原型是怎么组织的，方便后续继续扩展场景交互、接后端同步，或者让新开发者快速理解仓库。

## 当前技术选型

- 前端框架：`React`
- 场景层：`react-konva`
- 构建工具：`Vite`
- 自动验证：`Playwright`

当前不包含：

- 后端服务
- 实时同步
- 数据库存储
- 可视化场景编辑器

所以这版更准确地说，是一个“场景驱动的前端原型”。

## 分层结构

当前实现分成四层：

### 1. 页面与状态层

入口在：

- [src/main.jsx](/D:/workspace/mini-home/src/main.jsx)
- [src/App.jsx](/D:/workspace/mini-home/src/App.jsx)
- [src/components](/D:/workspace/mini-home/src/components)
- [src/hooks](/D:/workspace/mini-home/src/hooks)
- [src/scene](/D:/workspace/mini-home/src/scene)

这一层负责：

- 启动 React 应用
- 维护场景模式状态
- 控制信件浮层、调试模式、电视开关、引体帧循环
- 组织页面布局，包括顶部标题、场景区、右侧说明区和验证区
- 协调各个组件、hook 和纯逻辑模块

当前核心状态包括：

- `manifest`
- `currentMode`
- `tvOn`
- `debugVisible`
- `letterOpen`
- `pullupFrameIndex`
- `propPositions`

### 2. 场景渲染层

当前由以下文件协同完成，由 `react-konva` 承担：

- [src/components/SceneStage.jsx](/D:/workspace/mini-home/src/components/SceneStage.jsx)
- [src/components/SceneNode.jsx](/D:/workspace/mini-home/src/components/SceneNode.jsx)
- [src/components/DebugGrid.jsx](/D:/workspace/mini-home/src/components/DebugGrid.jsx)

这一层负责：

- 创建 Konva `Stage`
- 按 `z` 排序绘制背景、家具、状态图、道具和角色
- 给物件挂点击和拖拽能力
- 在调试模式下绘制网格、边框和资源 id

当前场景对象统一走 manifest 数据，不是在代码里硬编码每一张图的位置。

### 3. 资源与场景配置层

入口在：

- [assets/scene-manifest-v1.json](/D:/workspace/mini-home/assets/scene-manifest-v1.json)
- [assets](/D:/workspace/mini-home/assets)

这一层负责：

- 定义 Stage 尺寸和网格大小
- 定义背景资源
- 定义家具、道具、状态图、角色图的坐标、宽度、层级和交互属性
- 定义拖拽区域

可以把这份 manifest 理解成“首版客厅的场景输入”。

### 4. 验证层

分成两部分：

- 页面内验证：在 [src/scene/validation.js](/D:/workspace/mini-home/src/scene/validation.js) 内纯计算
- 自动验证脚本：在 [scripts/verify-scene.mjs](/D:/workspace/mini-home/scripts/verify-scene.mjs)

页面内验证负责：

- 检查关键物件是否可见
- 检查电视、信件板、单杠是否还在预期区域
- 检查单杠和信件板是否互相遮挡
- 检查零食和水杯是否还落在茶几有效区域

自动验证负责：

- 跑 `build`
- 启动 `preview`
- 打开页面
- 检查校验卡片是否通过
- 回归电视、信件板、单杠和拖拽四条主链路

## 运行时数据流

页面启动后的主流程是：

1. `App.jsx` 请求 `/assets/scene-manifest-v1.json`
2. 读取到 manifest 后，初始化道具位置
3. [src/hooks/useAssetImages.js](/D:/workspace/mini-home/src/hooks/useAssetImages.js) 按 manifest 中的资源路径加载图片
4. 根据当前状态计算：
   - 当前角色图
   - 当前电视状态图
   - 当前需要渲染的场景对象列表
5. [src/components/SceneStage.jsx](/D:/workspace/mini-home/src/components/SceneStage.jsx) 按 `z` 排序后渲染到 Konva Stage
6. 页面右侧同步显示当前互动说明和校验结果

## 交互流

### 电视

- 触发 `handleSceneInteraction("tv")`
- 切到 `currentMode = "tv"`
- 打开电视屏幕状态图
- 更新右侧说明区

### 信件板

- 触发 `handleSceneInteraction("letters")`
- 切到 `currentMode = "letters"`
- 打开 `letterOpen`
- 渲染信件浮层

### 单杠

- 触发 `handleSceneInteraction("pullup-bar")`
- 切到 `currentMode = "pullup"`
- 启动引体两帧循环

### 道具拖拽

- 拖动 `snack-bag` 或 `water-cup`
- 在 `handlePropDragEnd` 里根据拖拽区做边界限制
- 用 `clampToZone` 做轻网格吸附
- 更新 `propPositions`

## 目录与职责

### `src/`

- `main.jsx`：应用入口
- `App.jsx`：页面状态编排和主入口
- `styles.css`：整体样式
- `components/`：场景舞台、调试网格、单节点、信件浮层和右侧面板
- `hooks/`：运行时资源加载 hook
- `scene/`：布局计算、拖拽约束和页面内校验
- `noteCopy.js`：互动说明文案

### `assets/`

- `backgrounds/`：背景图
- `furniture/`：主要家具图
- `props/`：桌面小物
- `states/`：电视、信件等状态图
- `avatars/`：角色状态图
- `scene-manifest-v1.json`：场景配置

### `scripts/`

- `verify-scene.mjs`：自动场景冒烟验证

### `docs/`

- `首版美术素材清单与要求.md`
- `首版场景验证方案.md`
- `首版前端架构说明.md`

## 当前工程边界

当前这版前端原型故意把范围压得比较小：

- 只有一个固定客厅
- 只有少量可互动对象
- 没有后端
- 没有持久化保存
- 没有多用户同步

这样做的目的，是先把“场景感、互动感、验证方式”跑通。

## 后续扩展建议

后面如果继续做，可以优先沿这条线扩：

1. 把 `App.jsx` 里的状态和场景逻辑拆到独立模块
2. 给 manifest 增加更明确的交互配置字段
3. 把信件和物件留言接到真实存储
4. 增加更稳定的资源裁边和命中区域描述
5. 继续扩自动验证，覆盖更多场景状态
