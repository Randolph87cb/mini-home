## 文档目的

这份文档用于给后续接手 `mini-home` 的开发者一个明确的当前状态说明，重点回答四个问题：

1. 项目现在做到哪一步了
2. 哪些部分已经验证过
3. 当前已知问题是什么

## 当前结论

`mini-home` 现在已经具备继续开发的基础，并且已经不是“只能演示”的壳子，而是一个单设备本地可用的前端样板。

更准确地说，当前适合接手继续推进这些事情：

- 调整客厅布局和素材
- 扩展电视 / 单杠 / 信件板互动
- 继续整理前端状态结构
- 把当前本地留言数据接到后端
- 把当前本地保存升级为云端同步
- 继续扩自动验证

当前还不适合直接当成完整产品工程接手长期维护，因为后端、账号、同步和持久化都还没接。

## 当前已知问题

### 1. 当前仍是前端原型

现在仍然没有：

- 登录
- 双人绑定
- 云端留言存储
- 实时同步
- 数据库

所以这版还不能算完整产品，但已经不是纯场景演示，而是“本地可用、等待后端接入”的样板。

### 2. 前端结构刚完成第一轮拆分，状态编排仍集中在 `src/App.jsx`

当前已经把资源加载、场景节点、舞台组合和页面内校验从 `App.jsx` 中拆出，并接上本地持久化；但 `App.jsx` 仍然掌握主要状态编排。下一步继续接数据时，最好优先整理：

- 状态计算
- 留言与信件数据流
- 本地保存和远端同步接入点
- 测试 API 和业务状态的边界

### 3. 素材留白和命中区域仍可能影响布局精度

当前 AI 生成透明素材虽然已经能用于原型，但仍然存在：

- 留白偏大
- 视觉重心和图片外框不一致
- 点击命中区和视觉中心未完全分离

这意味着：

- 自动校验能防明显回归
- 但不能完全替代人工判断构图是否自然

### 4. manifest 仍偏“首版够用”

当前 `scene-manifest-v1.json` 已经能驱动原型，但还没有更细的结构化字段，例如：

- 更精确的命中区
- 更精确的视觉锚点
- 后续留言挂载点
- 更明确的交互配置

## 下一步建议先做什么

如果现在交给别人继续开发，我建议优先顺序是：

1. 把当前本地数据模型整理成可对接后端的接口层  
现在已经有短便签、长信和道具位置的本地模型，下一步应把它们收敛成稳定的服务接口。

2. 接入一个最小云端保存层  
当前已经有本地持久化，下一步应该让两个人能跨设备看到同一份内容。

3. 继续整理 `src/App.jsx` 的状态编排  
重点是把页面状态、数据状态和自动化测试暴露边界进一步分开。

4. 继续修素材和场景命中区  
这一步会直接影响后续点击、拖拽和布局体验。

5. 最后再接账号和双人同步  
不要在当前数据边界和同步策略还没定清楚前就急着上多人协作。
