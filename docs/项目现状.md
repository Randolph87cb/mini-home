## 当前结论

`mini-home` 现在已经具备继续开发的基础，但当前阶段仍然是“前端原型”，不是完整产品。

更准确地说，当前适合接手继续推进这些事情：

- 调整客厅布局和素材
- 扩展电视 / 单杠 / 信件板互动
- 继续拆前端结构
- 接入留言数据
- 接入本地保存或云端同步
- 继续扩自动验证

当前还不适合直接当成完整产品工程接手长期维护，因为后端、账号、同步和持久化都还没接。

## 当前已知问题

### 1. 当前仍是前端原型

现在没有：

- 登录
- 双人绑定
- 留言存储
- 实时同步
- 数据库

所以这版还不能算完整产品，只能算场景和交互原型。

### 2. `src/App.jsx` 仍然偏大

现在状态、场景渲染、交互、页面内校验基本都还集中在一个文件里。继续开发前，最好逐步拆分：

- 场景渲染
- 状态计算
- 校验逻辑
- 浮层 UI

### 3. 素材留白和命中区域仍可能影响布局精度

当前 AI 生成透明素材虽然已经能用于原型，但仍然存在：

- 留白偏大
- 视觉重心和图片外框不一致
- 点击命中区和视觉中心未完全分离

这意味着：

- 自动校验能防明显回归
- 但不能完全替代人工判断构图是否自然

### 4. manifest 仍偏“首版够用”

当前 `scene-manifest-v1.json` 已经能驱动原型，但还没有更细的结构化字段，例如：

- 更精确的命中区
- 更精确的视觉锚点
- 后续留言挂载点
- 更明确的交互配置

## 下一步建议先做什么

如果现在交给别人继续开发，我建议优先顺序是：

1. 把 `src/App.jsx` 拆模块  
先把场景渲染、校验、交互和浮层分开，否则后面接数据会越来越重。

2. 明确留言数据模型  
先定短留言、长信件、物件挂载点的数据结构，再谈后端。

3. 接一个最小持久化层  
哪怕先本地保存，也比所有状态只停留在内存里更适合继续开发。

4. 继续修素材和场景命中区  
这一步会直接影响后续点击、拖拽和布局体验。

5. 最后再接账号和双人同步  
不要在当前结构还没拆清楚前就急着上多人协作。
