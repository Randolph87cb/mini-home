# 首版美术素材清单与要求

## 目的

这份文档用于指导 `线上小家` 首版客厅场景的美术制作。

目标不是产出一张完整插画，而是产出一组可以在网页场景里独立摆放、互动、切状态、支持拖拽扩展的素材。

## 统一风格要求

- 视角：正面视角。
- 风格：温柔绘本感，偏生活化，不要像素风，不要重二次元。
- 气质：安静、柔和、有共同生活痕迹，不做强游戏感。
- 色调：奶油白、浅木色、雾粉、浅绿、浅灰蓝。
- 光线：室内暖光，整体均匀柔和。
- 线条：轻手绘感，边缘不要太硬。
- 细节密度：中低，不要做成复杂装饰插画。

## 统一制作规格

- 基准场景尺寸：`1920 x 1080`。
- 所有单物件素材按同一基准比例绘制。
- 运行时素材默认使用透明底。
- 可编辑源文件必须保留分层。
- 推荐保留源文件格式：`PSD`、`Figma`、`Procreate` 等可分层格式。
- 推荐运行时格式：
  - 静态背景：`WebP` 或 `PNG`
  - 单物件：透明 `PNG`
  - 简单帧动画：多张 `PNG` 或后续合图
- 阴影规则：优先将柔和阴影烘焙进素材；只有明确需要动态变化的阴影再拆成独立层。

## 制作原则

- 不要把整个房间画成一张不可拆分的大图。
- 不要把可互动物件直接烘焙进背景。
- 不要把以后可能变化的桌面小物和桌子主体画死在一起。
- 角色首版先按状态图制作，不先做复杂骨骼动画。
- 信件板和信件内容必须拆开，不能是一张死图。

## 素材分组

### 1. 背景组

用途：承载固定不动的客厅空间基础。

包含：

- 墙面
- 地板
- 窗户或基础墙面装饰
- 固定环境光影

要求：

- 不包含电视、沙发、单杠、茶几、信件板。
- 地面和墙面边界清楚。
- 要有“家里有人住”的温和氛围，但不能太满。

交付建议：

- `backgrounds/living-room-base.webp`

### 2. 固定核心物件组

用途：场景主体和互动核心。

包含：

- 沙发
- 电视机身
- 单杠
- 茶几
- 信件板

要求：

- 每个物件单独透明底。
- 所有物件透视关系一致。
- 同一批素材的光影和边缘风格一致。

### 3. 桌面小物组

用途：增强生活感。

包含：

- 零食
- 水杯

要求：

- 单独透明底。
- 比例相对茶几真实自然。
- 不要过度细碎。

### 4. 角色组

用途：表达陪伴感和互动反馈。

首版状态：

- 两个人一起坐在沙发上，她靠着你
- 看电视状态
- 你做引体状态
- 看信状态

要求：

- 首版优先使用整张状态图。
- 角色缩小后仍要能看清姿态。
- 情绪是放松、安稳，不做夸张表演。

### 5. 互动状态组

用途：支持互动反馈和动画切换。

包含：

- 电视屏幕关闭态
- 电视屏幕开启态
- 信封关闭态
- 信封展开态
- 引体动作的补帧

要求：

- 状态切换前后要有明显对应关系。
- 电视屏幕内容层和电视机身分开。
- 信封展开态要能自然过渡到阅读态。

### 6. UI 装饰组

用途：承载留言、信件和节目内容的界面装饰。

包含：

- 信纸底图
- 节目卡片底图
- 小贴条底图
- 图钉、胶带、纸角装饰

要求：

- 归属于界面层，不混入场景层。
- 风格与客厅一致。
- 不要做成太模板化的手账风。

## 单素材执行要求

### `living-room-base`

- 只包含固定空间。
- 不包含任何未来需要单独互动的核心物件。
- 可以有基础窗户、墙面留白和柔和氛围。

### `sofa-main`

- 视觉上要能承接双人一起坐。
- 轮廓柔和，有包裹感。
- 靠枕如未来可能变化，可单独留层。

### `tv-body`

- 电视外框单独成图。
- 屏幕区域清晰，方便叠加节目内容或亮屏状态。
- 点击热点主要集中在电视正面区域。

### `tv-screen-off` / `tv-screen-on`

- 与 `tv-body` 对齐。
- 保持相同透视和边界。
- 开启态不需要真实节目截图，只要有“在播放”的氛围。

### `pullup-bar`

- 更像家里的单杠，不要做成健身房大型器械。
- 留出人物悬挂和动作的位置。

### `coffee-table`

- 台面留白足够摆放零食和水杯。
- 不建议把小物直接画死进桌面主体。

### `snack-bag` / `water-cup`

- 小物风格与整体一致。
- 尺寸不要抢戏。

### `letter-board`

- 更像生活里的挂板，不像公告栏。
- 可以有木质感、图钉、夹子，但不要繁杂。
- 必须和信件素材拆开。

### `letter-closed` / `letter-open`

- 关闭态适合挂在信件板上。
- 展开态适合进入阅读浮层。
- 两个状态在颜色和纸张特征上要有连续性。

### `avatar-sit-together`

- 默认状态。
- 你们坐在沙发上，她靠着你。
- 重点是关系感，不是复杂动作。

### `avatar-pullup-01` / `avatar-pullup-02`

- 保证动作循环自然。
- 单杠位置和手部动作要匹配。
- 第一版两帧即可。

### `avatar-read-letter`

- 姿势放松。
- 重点是“停下来读一封信”的状态。

## 自检清单

在交付每个素材前，逐项检查：

- 是否符合正面视角
- 是否符合统一配色和光线
- 是否保留源文件分层
- 是否输出透明底
- 是否方便单独摆放
- 是否方便点击区域定义
- 是否误把可互动内容画死在背景里
- 是否和其他素材比例一致
- 是否已经按命名规范整理

## 首版完成判定

当以下内容全部具备时，可以开始进入程序挂载阶段：

- 客厅背景完成
- 五个核心物件完成
- 两个桌面小物完成
- 角色默认状态和两类互动状态完成
- 电视和信件的状态图完成
- 至少一套基础 UI 装饰完成
